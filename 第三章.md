### 1.试述SQL的特点

**综合统一**：集数据定义（DDL）、数据操纵（DML）、数据控制（DCL）功能于一体。

**高度非过程化**：用户只需提出“做什么”，而无需指明“怎么做”，存取路径的选择由系统自动完成。

**面向集合的操作方式**：操作对象和结果都是元组的集合，而非单条记录。

**以同一种语法结构提供两种使用方式**：既是自含式语言（直接交互使用），又是嵌入式语言（嵌入到高级语言如 C++, Java 中）。

**语言简洁，易学易用**：核心功能仅用很少的动词即可完成（如 SELECT, INSERT, UPDATE, DELETE）。

### 2.说明在 DROP TABLE 时，RESTRICT 和 CASCADE 的区别

**RESTRICT（限制）**：这是默认选项。如果该表被其他对象引用，系统将**拒绝执行**删除操作。只有当该表没有被任何视图、约束、触发器等引用时，才能成功删除。

**CASCADE（级联）**：删除该表的同时，**自动删除**所有引用该表的对象（如依赖该表的视图）。如果删除了主表，相关的外键约束也会被一并清理。

### 3. 写出与关系代数等价的 SQL 表达式

已知关系 $S(A, B, C, D)$ 和 $T(C, D, E, F)$：$S()$

① $\sigma_{A=10}(S)$

```sql
SELECT * FROM S WHERE A = 10;
```

② $\Pi_{A,B}(S)$

```sql
SELECT DISTINCT A, B FROM S; 
-- 注：关系代数默认去重，SQL需加DISTINCT以严格等价
```

③ $S \Join T$（自然连接）

自然连接会自动匹配同名属性（即 C 和 D），并去掉重复列。

```sql
SELECT S.A, S.B, S.C, S.D, T.E, T.F 
FROM S NATURAL JOIN T;
-- 或者显式表达：
SELECT S.A, S.B, S.C, S.D, T.E, T.F 
FROM S, T 
WHERE S.C = T.C AND S.D = T.D;
```

④ $S \Join_{S.C=T.C} T$（等值连接）

```sql
SELECT * FROM S JOIN T ON S.C = T.C;
-- 或者：
SELECT * FROM S, T 
WHERE S.C = T.C;
```

⑤ $S \Join_{A < E} T$（不等连接）

```sql
SELECT * FROM S JOIN T ON S.A < T.E;
-- 或者：
SELECT * FROM S, T 
WHERE S.A < T.E;
```

⑥ $\Pi_{C,D}(S) \times T$（投影后做笛卡尔积）

```sql
SELECT S.C, S.D, T.C, T.D, T.E, T.F 
FROM S, T;
-- 注意：SQL中的逗号或 CROSS JOIN 即代表笛卡尔积
```

### 4. 建立 SPJ 数据库的 4 个表

在建表时，我们需要定义主键（Primary Key）和外键（Foreign Key）来保证数据的完整性。

```sql
-- 1. 供应商表 S
CREATE TABLE S (
    SNO CHAR(5) PRIMARY KEY,
    SNAME CHAR(20) UNIQUE,
    STATUS INT,
    CITY CHAR(20)
);

-- 2. 零件表 P
CREATE TABLE P (
    PNO CHAR(5) PRIMARY KEY,
    PNAME CHAR(20),
    COLOR CHAR(10),
    WEIGHT INT
);

-- 3. 工程项目表 J
CREATE TABLE J (
    JNO CHAR(5) PRIMARY KEY,
    JNAME CHAR(20),
    CITY CHAR(20)
);

-- 4. 供应情况表 SPJ
CREATE TABLE SPJ (
    SNO CHAR(5),
    PNO CHAR(5),
    JNO CHAR(5),
    QTY INT,
    PRIMARY KEY (SNO, PNO, JNO),
    FOREIGN KEY (SNO) REFERENCES S(SNO),
    FOREIGN KEY (PNO) REFERENCES P(PNO),
    FOREIGN KEY (JNO) REFERENCES J(JNO)
);
```

### 5. 针对 SPJ 数据库的 SQL 操作

这里“上海产”通常指供应商在上海。

```sql

```

这是一个“否定”逻辑，建议使用 `NOT IN` 或 `NOT EXISTS`。

```sql
SELECT JNO FROM J
WHERE JNO NOT IN (
    SELECT DISTINCT JNO FROM SPJ, S
    WHERE SPJ.SNO = S.SNO AND S.CITY = '天津'
);
```

### 6. 基本表与视图的区别和联系

- **基本表 (Base Table)**：实际存储在数据库中的表，是“实表”，占用物理存储空间。
- **视图 (View)**：从一个或多个基本表（或其他视图）导出的虚拟表。它只保存 **定义（SQL 查询语句）**，不保存实际数据，是“虚表”。

| **特征**   | **基本表**             | **视图**                           |
| ---------- | ---------------------- | ---------------------------------- |
| **存储**   | 存储表结构和物理数据   | 仅存储定义，数据随基本表动态变化   |
| **持久性** | 永久存在，直到显式删除 | 逻辑存在，依赖于基础表             |
| **联系**   | 视图的数据来源于基本表 | 基本表数据的改变会即时反映在视图中 |

### 7. 试述视图的优点

1. **简化用户操作**：将复杂的连接查询、过滤条件封装在视图中，用户只需进行简单查询。
2. **安全性**：可以限制用户只能访问表中的特定行或列（例如隐藏薪资列）。
3. **逻辑独立性**：如果基本表的结构发生变化（如拆分表），只需修改视图定义，而不必修改应用程序。
4. **多角度展示**：同一数据可以根据不同用户的需求定义为不同的视图。

### 8. 视图的更新性

**规则**：并非所有视图都是可更新的。基本原则是：**如果对视图的更新不能唯一地转换成对基本表的更新，则该视图不可更新。**

- **可更新视图**：通常是从单个基本表导出的，且包含了主键的视图。

  - *例*：`CREATE VIEW V_S1 AS SELECT * FROM S WHERE STATUS > 10;`

- **不可更新视图**：

  1. 包含 **聚合函数**（SUM, AVG, COUNT 等）。
  2. 包含 **DISTINCT**、**GROUP BY** 或 **HAVING** 子句。
  3. 由多个表 **连接（Join）** 形成的视图（部分数据库有限制支持）。

  - *例*：`CREATE VIEW V_AVG AS SELECT JNO, AVG(QTY) FROM SPJ GROUP BY JNO;`（无法通过平均值反推单条记录 QTY）

### 10. 空值 (NULL)

- **什么是空值**：空值表示“不知道”、“不存在”或“无意义”的值。它不等于 0，也不等于空字符串 `''`。
- **举例**：一个新入职员工的“提成”字段可能为空，因为还没产生销售额。
- **SQL 表示**：使用关键字 `NULL`。
- **运算规则**：
  1. **比较运算**：必须使用 `IS NULL` 或 `IS NOT NULL`。使用 `= NULL` 结果永远为“未知”（Unknown）。
  2. **算术运算**：`NULL` 与任何数加减乘除，结果都是 `NULL`。
  3. **聚合函数**：`COUNT(*)` 会包含空值行，但 `SUM`、`AVG`、`COUNT(列名)` 等会**自动跳过**空值。
