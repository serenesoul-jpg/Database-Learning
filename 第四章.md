# 习题四

### 1. 什么是数据库的安全性？

数据库的安全性是指保护数据库以防止不合法的使用所造成的**数据泄露、更改或破坏**。其核心目标是确保只有授权用户才能访问合法数据。

### 2. 举例说明对数据库安全性产生威胁的因素。

- **非授权用户恶意存取**：黑客攻击、内部人员越权访问。
- **敏感数据泄露**：数据在传输过程中被窃听。
- **数据完整性破坏**：非法篡改财务数据、考试成绩等。
- **物理环境威胁**：服务器硬件被盗、自然灾害导致存储介质损坏。

### 3. 实现数据库安全性控制的常用方法和技术。

- **用户身份鉴别**：静态口令、生物特征识别、智能卡。
- **存取控制**：确定用户权限（GRANT/REVOKE）。
- **视图机制**：隐藏不应让用户看到的敏感数据。
- **审计（Audit）**：记录用户的所有操作以便事后追溯。
- **数据加密**：存储加密和传输加密。

### 4. 什么是自主存取控制（DAC）和强制存取控制（MAC）？

- **自主存取控制 (Discretionary Access Control, DAC)**：
  - **定义**：用户可以对特定数据对象行使某些权力，且可以将这些权力转授给其他用户。
  - **特点**：非常灵活，但安全性相对较低，难以防止“特洛伊木马”式攻击。
- **强制存取控制 (Mandatory Access Control, MAC)**：
  - **定义**：系统为每个数据对象和每个用户分配安全标记（如：绝密、机密、公开）。只有满足特定规则（如用户级别 $\ge$ 数据级别）时才能访问。
  - **特点**：由系统强制执行，用户不能私自转让权限，安全性极高。

### 5. SQL 的 GRANT 授权练习

针对学生（Student）和班级（Class）两个表的授权操作如下：

① 授予用户 U1 对两个表的所有权限，并可给其他用户授权

```sql
GRANT ALL PRIVILEGES 
ON TABLE 学生, 班级 
TO U1 
WITH GRANT OPTION;
```

② 授予用户 U2 对“学生”表具有查看权限，对“家庭住址”具有更新权限

```sql
GRANT SELECT, UPDATE(家庭住址) 
ON TABLE 学生 
TO U2;
```

③ 将对“班级”表查看权限授予所有用户

```sql
GRANT SELECT 
ON TABLE 班级 
TO PUBLIC;
```

④ 将对“学生”表的查询、更新权限授予角色 R1

```sql
GRANT SELECT, UPDATE 
ON TABLE 学生 
TO R1;
```

⑤ 将角色 R1 授予用户 U1，并且 U1 可继续授权给其他角色

```sql
GRANT R1 
TO U1 
WITH ADMIN OPTION; 
-- 注：角色的转授权使用 WITH ADMIN OPTION，对象的转授权使用 WITH GRANT OPTION
```



### 6 & 7. 授权 (GRANT) 与 撤销 (REVOKE) 练习

针对职工表 (Emp) 和部门表 (Dept)，相应的 SQL 如下：

① 用户王明对两个表有 SELECT 权限

- **授权：** `GRANT SELECT ON TABLE 职工, 部门 TO 王明;`
- **撤销：** `REVOKE SELECT ON TABLE 职工, 部门 FROM 王明;`

② 用户李勇对两个表有 INSERT 和 DELETE 权限

- **授权：** `GRANT INSERT, DELETE ON TABLE 职工, 部门 TO 李勇;`
- **撤销：** `REVOKE INSERT, DELETE ON TABLE 职工, 部门 FROM 李勇;`

③ 每个职工只对自己的记录有 SELECT 权限

这需要用到 **视图 (View)** 和系统变量（如 `USER`）。

- **授权：**

  ```sql
  CREATE VIEW 个人记录 AS 
  SELECT * FROM 职工 WHERE 姓名 = USER; -- 假设系统变量USER返回当前用户名
  GRANT SELECT ON 个人记录 TO PUBLIC;
  ```

- **撤销：** `DROP VIEW 个人记录;` (或 `REVOKE SELECT ON 个人记录 FROM PUBLIC;`)

④ 用户刘星对职工表有 SELECT 权限，对“工资”字段具有更新权限

- **授权：** `GRANT SELECT, UPDATE(工资) ON TABLE 职工 TO 刘星;`
- **撤销：** `REVOKE SELECT, UPDATE(工资) ON TABLE 职工 FROM 刘星;`

⑤ 用户张新具有修改这两个表的结构的权限

- **授权：** `GRANT ALTER ON TABLE 职工, 部门 TO 张新;`
- **撤销：** `REVOKE ALTER ON TABLE 职工, 部门 FROM 张新;`

⑥ 用户周平具有对两个表的所有权限，并具有给其他用户授权的权限

- **授权：** `GRANT ALL PRIVILEGES ON TABLE 职工, 部门 TO 周平 WITH GRANT OPTION;`
- **撤销：** `REVOKE ALL PRIVILEGES ON TABLE 职工, 部门 FROM 周平 CASCADE;`

⑦ 用户杨兰具有从每个部门职工中 SELECT 最高/最低/平均工资的权限，但不能查看个人工资

这必须通过 **视图** 隐藏明细数据。

- **授权：**

  ```sql
  CREATE VIEW 部门工资统计 AS
  SELECT 部门号, MAX(工资) AS 最高, MIN(工资) AS 最低, AVG(工资) AS 平均
  FROM 职工 GROUP BY 部门号;
  GRANT SELECT ON 部门工资统计 TO 杨兰;
  ```

- **撤销：** `REVOKE SELECT ON 部门工资统计 FROM 杨兰; DROP VIEW 部门工资统计;`

### 8. 强制存取控制 (MAC) 中的核心概念

在 MAC 模型中，系统通过比较“安全标记”来决定访问权限：

- **主体 (Subject)**：系统中的活动实体，既包括实际用户，也包括代表用户的进程。
- **客体 (Object)**：系统中的被动实体，受主体操纵。在数据库中指模式、表、属性列、元组等。
- **敏感度标记 (Sensitivity Label)**：对于主体，称为**许可证级别**；对于客体，称为**密级**。常见的级别如：公开 (Public) < 机密 (Secret) < 绝密 (Top Secret)。

### 9. MAC 如何确定主体能否存取客体？

MAC 遵循“向下读，向上写”的规则（以 Bell-LaPadula 模型为例）：

1. **读操作 (Read)**：主体的许可证级别 **大于或等于** 客体的密级时，才能读取。防止机密信息向下泄露。
2. **写操作 (Write)**：主体的许可证级别 **小于或等于** 客体的密级时，才能写入。防止高级别信息被写入低级别区域。

> **简言之：** 一个只有“机密”权限的人，可以看“公开”数据（向下读），但不能看“绝密”数据；他写的数据密级至少应为“机密”。

### 10. 什么是数据库审计 (Audit)？

**定义：**

审计功能是指 DBMS 的审计模块在用户操作数据库时，将所有的操作记录在 **审计日志 (Audit Log)** 中。

**为什么要提供审计功能？**

1. **可追溯性**：当发生安全违规（如数据被非法篡改或窃取）时，可以通过日志找出罪魁祸首。
2. **威慑作用**：用户知道自己的行为被记录，会减少恶意操作的动机。
3. **发现隐患**：通过分析审计日志，可以发现系统中潜在的安全漏洞或异常访问模式。
4. **合规性**：许多行业标准（如金融、医疗）强制要求记录数据访问日志以备核查。
